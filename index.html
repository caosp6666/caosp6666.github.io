从0开始Hexo博客搭建

Posted on 2020-06-28

1.安装Node.js

官网下载安装即可

2.配置淘宝cnpm镜像源

npm install -g cnpm --registry==https://registry.npm.taobao.org

再通过cnpm -v查看是否配置成功

3.安装hexo-cli

cnpm install -g hexo-cli

再通过hexo -v查看是否安装成功

4.hexo init！

首先创建一个Myblog文件夹（之后所有相关文件都会在这里，不用的时候直接全部删掉即可）并进入目录

5. 初步使用

启动博客服务器hexo s-> 访问localhost:4000即可看到hexo的初始化页面，默认创建了一篇helloworld文章。（这个命令通常是作为博客写文章后预览使用）

进入D:\myblog\source\_posts目录即可看到这个文章的markdown文件。

下面创建自己的第一篇文章hexo n "起点" 此时_posts目录中就生成了一个新的.md文件。然后编辑这个文件即可写文章了。写好保存后，在==blog目录==运行以下命令

6. 部署到Github

  1. 在github上建立一个新仓库，命名为username.github.io （必须这样命名！）其他选项默认即可

  2. 在blog目录中打开命令行

  3. 修改配置文件_config.yml

  4. 回到blog下的命令行执行hexo d 即可完成部署

  5. 访问 username.github.io 测试

7. 开始blog之路

以后写新博文时候进行以下操作即可

8. 主题更换

选择了GitHub上star最多的一个主题next，项目地址： next

在blog目录下，执行D:\myblog>git clone https://github.com/next-theme/hexo-theme-next.git themes/next 便将主题下载到了themes文件夹下

然后修改_config.yml 文件中的theme为next 再重新部署即可

主题的配置请参考主题项目的Document

参考与引用

www.codesheep.cn

https://theme-next.js.org/

图床是什么

图床即储存图片的服务器。

Markdown的本质是对html的解析，于是就存在内置图片的来源问题。在本地编写的md文件，可以插入本地的图片，或者配置远程url。而想讲文章发布到博客给别人看就必须把本地图片上传到图床，再把生成的url复制到文章中（如果图片多，会是一个很耗时耗力的过程）
